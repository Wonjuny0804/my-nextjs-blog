프로그래밍을 대학교에서 처음 공부했을때 C라는 언어 대해서 학습했다. 가장 처음에 책을 펴면 컴퓨터의 작동원리에 대해서 조금 나오고 그 다음에 컴파일 언어와 인터프리터 언어에 대한 차이점이 나온다.
조금만 더 읽어가다보면 표현식 그리고 문에 대한 글이 아주 짧게 나오고 나서 바로 변수에 대한 부분이 나오던데 이제 보니 다른 언어들도 레파토리가 비슷한 것 같다.

그런 이유인즉, 결국 프로그래밍 언어도 하나의 "언어", 일종의 규칙이 있는 기호이기 때문일 것이다. 예전에 자바스크립트를 정리하면서 여러 책들과 블로그 글들을 참고했는데 거기서도 이 표현식과 문에 대해서 많이들 얘기하고 있었다.

여러가지 참고자료들이 있었는데 그 중에서 오늘은 유명한 코뿔소책 과 mdn, 모던자바스크립트 딥다이브를 참고해서 작성을 하였다.

## 표현식(Expression)과 문장(statement), 연산자(Operator)

자바스크립트 언어의 문법을 이해할 때는 이 언어의 문법이 표현식과 문장(statement)로 나누어진다는 것을 알 필요가 있다. 이 둘을 이해할 수 있다면 많은 그들에서는 자바스크립트로 코드를 짤때 더 간결하고 이쁜 코드를
쓸 수 있다고 말하고 있다.

그렇다면 많은 글들에서 말하고 있는 표현식과 문장에 대해서 알아보자.

> In general, an expression is a snippet of code that evaluates to a value. A statement is a snippet of code that performs an action. Wherever JavaScript expects a statement, you can write an expression. - masteringJS -

> 표현식의 정의는 값으로 평가될 수 있는 문장을 말한다. 값으로 평가될 수 있다는 뜻은 여러가지 뜻을 내포하고 있지만 변수에 할당할 수 있다는 뜻이 되기도 하고 함수의 인자값으로 전달이 가능하다는 뜻이다. - 모던 자바스크립트 딥다이브 -

> 표현식이 아닌 문(statement)은 일종의 지시라고 생각할 수 있고, 표현식은 무언가를 요청하는 것이라고 생각할 수 있습니다. - 러닝 자바스크립트 -

결국 표현식이란 값(value)로 귀결되는 코드 부분이라고 볼 수 있을 것 같다. 그렇다면 값이라는 것은 무엇을 뜻할까? 그건 앞으로 계속해서 생각을 해보도록 하자. 그리고 문장은 어떠한 action을 하는 코드라고 생각하면 된다고 하는데, 지금으로써는 자바스크립트에서 표현식은
**값으로 평가될 수 있는 문장**이라고 기억하도록하자.

```ts
true;
("This is a Expression");
"WonjunDev" + "." + "tech";
```

위의 코드들은 모두 표현식의 예시이다. 하나의 코드 덩어리지만 우리가 자바스크립트에서 알고 있는 "값"이기도 하기 때문이다.

웹에서는 문장은 어떠한 action을 하는 코드 덩어리를 문장이라고 했다. 그렇다면 일련의 행동을 하는 것들이 문장이라면 이 문장은 표현식으로 이루어져 있다고 할 수도 있을 것이다. 그리고 이 표현식들로 이루어진 문장들이 뭉쳐서
하나의 프로그램이 된다고 우리는 말할 수 있다.

### 연산자는 표현식의 '명사'에 대한 '동사'

> 표현식이 값이 되는 것이라면 연산자는 값을 만드는 행동이라는 뜻 - 러닝 자바스크립트 -

표현식은 값으로 평가 될 수 있는 문장이라고 했다. 따라서 이 표현식을 어딘가에 쓸 수 있고 그러기 위해서 여러가지 연산자를 자바스크립트에서 제공을 하는데 우리가 이미 알고 있는 사칙연산 연산자, 논리 연산자, 비교 연산자가 이에 해당한다.

## 데이터 타입의 두 가지

어느 프로그래밍 언어가 그렇겠지만, 데이터에는 타입이 존재한다. 자바스크립트에서 값은 크게 두 가지로 나뉠 수 있는데 바로 **원시타입(Primitive type)** 과 **객체타입(Object type)** 이다.

이 둘은 성격이 조금은 다른 데이터 타입인데 이를 잘 이해하는 것이 자바스크립트 언어를 이해하는데 많은 도움을 준다. 원시타입은 다음과 같은 특징을 가진다.

- 원시타입은 불변(immutable)이다.

여기서 말하는 불변은 값을 바꿀수없는 것과 조금 다른 것을 의미한다. 이 부분을 이해하려면 우리는 값을 할당한다의 의미에 대해서 더 깊게 들어가야 한다.

만약 우리가 자바스크립트에서 다음과 같은 코드를 썼다고 생각해보자.

```js
const name = "Wonjun Jang";
```

우리는 name이라고 하는 식별자에 Wonjun Jang이라고 하는 문자열 값을 할당했다. 컴퓨터의 작동원리 상 "Wonjun Jang"이라고 하는 문자열 값은 이진법으로 계산이 되어서 물리적으로 어딘가에 기억이 되어야할 것이다.

Wonjun Jang이라고 하는 영어문자를 이진법으로 바꾸어서 메모리 어딘가에 저장하고 그 주소 부분과 name이라고하는 식별자를 서로 연결하는 과정을 우리는 "값을 할당"했다고 표현하고 있는 것이다.

그렇다면, 우리는 값의 할당을 한 `Wonjun Jang`을 고칠 수 있을까? 물론 고칠 수 있다. 지금은 `const`라고 하는 상수 선언 키워드로 만들었지만 `let`을 쓰거나 `var`를 쓴다면 가능한 일이다.

그렇지만 불변값이라는 것은 무슨 뜻일까? 그 뜻은 `Wonjun Jang`을 우리가 `Wonjun Jang Dev blog`라는 문자열로 고친다고 했을때 같은 메모리 주소에 더 길게 추가되는 것이 아니라

원래 있던 메모리 주소는 쓰지 않고 다른 메모리 주소를 할당해서 그곳에 `Wonjun Jang Dev blog`라는 값을 저장한다는 뜻이다.

우리는 데이터의 타입에 대해서 얘기하면서 값의 할당에 대해서도 얘기를 했는데 아마도 많은 이들이 알고 있겠지만 자바스크립트의 흥미로운 사실은 메모리에 공간을 확보해서 값을 넣어주고 서로 연결지어줄 때 자바스크립트만의 독특한 특징인 `Hoisting` 덕분에
이 값들은 `undefined`로 초기화 된다는 사실이다.

왜 이렇게 만들었는지는 지금으로써는 알아내기 힘들지만, C언어와 C++같은 언어들을 공부했던 사람들은 컴파일 언어의 경우 프로그램의 컴파일 전 이 프로그램이 필요로 하는 메모리 공간의 크기를 미리 알아야함을 알 것이다. 따라서 이 부분도 자바스크립트 프로그램의 메모리
할당 크기를 얼추 estimate하기 위해서 내부적으로 짜여진게 아닌가 하는 생각이 들긴한다.

마지막으로 객체에 대한 이야기를 하고 오늘의 이야기의 결론을 내려고 한다. 객체는 다양한 값과 형태를 가질 수 있다. 그리고 그 본질은 컨테이너라고 많은 레퍼런스들에서 말해준다.

> 컨테이너의 내용물은 시간이 지나면서 바뀔 수 있지만, 내용물이 바뀐다고 컨테이너가 바뀌는 건 아닙니다. 즉, 여전히 같은 객체입니다. - 러닝 자바스크립트 -

객체에는 많은 것들을 담을 수 있고 자바스크립트에서는 꽤 유용하게 쓰인다. 자바스크립트 객체 안의 요소는 "프로퍼티"라고 하는 이름으로 불리고 우리가 아닌 key value로 이루어진 데이터 형식이다.

```js
const testObject = {
  property1: "yes",
  property2: true,
  property3: {
    name: "Wonjun",
    age: undefined,
  },
};
```

위의 예시에서 보듯이 정말 key, value의 형태라면 어떠한 것이든 가질 수 있는게 객체 타입이라서 무엇인가를 저장하고 찾아야할때 또한 상태를 저장할때 유용하게 쓰인다.
또한 안의 값이 바뀌어도 재할당이 이루어지지 않기 때문에 const로 선언해도 무방하다.

자바스크립트라는 언어에서 가볍지만, 이해하면 정말 많은 도움이 되는 두가지에 대해서 알아보았다. 이 주제에 대해서 더 많이 알게 된다면 이 포스트를 업데이트할 것이다.
